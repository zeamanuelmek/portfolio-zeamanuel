# Theme Customization

SemEnaWerk's theming system lets you rebrand the entire UI by changing **a single function call**.
No digging through CSS files. No find-and-replace hex codes. Just configure and deploy.

---

## How It Works

The theme flows through three layers:

```
tokens.ts → createClientTheme() → MantineProvider → All Components
```

1. **`themes/tokens.ts`** — raw design tokens (colors, fonts, spacing)
2. **`themes/createClientTheme.ts`** — generates a complete Mantine theme with tint/shade color scales
3. **`MantineProvider`** — applies the theme to every component in the tree

---

## Quick Start: 5-Minute Theme Swap

### Step 1 — Create a Client Theme

```typescript
// themes/clients/startup-x.ts
import { createClientTheme } from '../createClientTheme';

export const startupXTheme = createClientTheme({
  brandColor: '#FF6B35',    // Client's primary brand color
  fontFamily: 'Poppins',    // Optional: override heading + body font
});
```

That's it. `createClientTheme` automatically generates:
- A **10-shade color scale** from your brand color (light → dark)
- Matching **component defaults** (button colors, focus rings, active states)
- Proper **contrast ratios** for text on colored backgrounds

### Step 2 — Apply in Layout

```typescript
// app/layout.tsx
import { MantineProvider } from '@mantine/core';
import { startupXTheme } from '@/themes/clients/startup-x';

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <MantineProvider theme={startupXTheme}>
          {children}
        </MantineProvider>
      </body>
    </html>
  );
}
```

### Step 3 — Deploy

Every component automatically picks up the new theme. Buttons turn orange, focus rings
match, cards glow in the new brand color. **Zero component changes needed.**

## The `createClientTheme` API

```typescript
createClientTheme({
  brandColor: string,     // Required — primary brand color (hex)
  fontFamily?: string,    // Optional — overrides both heading and body fonts
})
```

### What It Generates

| Theme Key | Generated From | Example |
|-----------|---------------|---------|
| `colors.brand` | 10-shade scale from `brandColor` | `brand.0` (lightest) → `brand.9` (darkest) |
| `colors.gold` | Ethiopian gold scale | Always available as accent |
| `colors.rust` | Ethiopian rust scale | For warnings/destructive |
| `colors.forest` | Ethiopian forest scale | For success states |
| `colors.earth` | Ethiopian earth scale | For neutral warmth |
| `primaryColor` | Set to `'brand'` | Components default to brand color |
| `fontFamily` | `fontFamily` param or Inter | Body text font |
| `headings.fontFamily` | `fontFamily` param or Space Grotesk | Heading font |
| `defaultRadius` | `'lg'` | Rounded, friendly aesthetic |
| `other.goldGradient` | Fixed | `linear-gradient(135deg, #D4AF37, #E8C84A)` |
| `other.creamBg` | Fixed | `#FAF8F3` |
| `other.goldGlow` | Fixed | Gold box-shadow string |

### Color Scale Generation

`createClientTheme` generates real tint/shade scales — not opacity hacks:

```
brand.0  →  lightest tint (backgrounds, hover states)
brand.1  →  light tint
brand.2  →  ...
brand.5  →  base color (your brandColor lands here)
brand.6  →  primary shade (default for filled components)
brand.7  →  darker shade
brand.9  →  darkest shade (text on light backgrounds)
```

## Advanced: Multi-Tenant Runtime Theming

For SaaS platforms serving multiple tenants, themes can be swapped at runtime:

```typescript
// Dynamic theme based on tenant
const tenantThemes = {
  'startup-a': createClientTheme({ brandColor: '#FF6B35' }),
  'startup-b': createClientTheme({ brandColor: '#6366F1' }),
  'startup-c': createClientTheme({ brandColor: '#059669' }),
};

function App({ tenantId }: { tenantId: string }) {
  const theme = tenantThemes[tenantId] ?? createClientTheme();

  return (
    <MantineProvider theme={theme}>
      <YourApp />
    </MantineProvider>
  );
}
```

No rebuild required. No CSS overrides. Just swap the theme object.
